## Glossary Terms

<GlossaryCard 
  term="API Key"
  category="Security"
  definition={
    <>
      A unique identifier used to authenticate API requests to Maxim AI. Keep your API keys secure and never share them publicly. API keys are typically included in request headers as <code>Authorization: Bearer YOUR_API_KEY</code>.
    </>
  }
/>

<GlossaryCard 
  term="Authentication"
  category="Security"
  definition={
    <>
      The process of verifying your identity when making API requests, typically done using an <strong>API key</strong> in the request headers. Authentication ensures that only authorized users can access Maxim AI services.
    </>
  }
/>

<GlossaryCard 
  term="Access Control"
  category="Security"
  definition={
    <>
      Security features that control who can access specific resources or perform certain actions. Maxim AI implements role-based access control (RBAC) to manage permissions for:
      <ul style={{ marginTop: '0.5rem', marginBottom: '0', paddingLeft: '1.5rem' }}>
        <li>API key management</li>
        <li>Workspace access</li>
        <li>Resource permissions</li>
      </ul>
    </>
  }
/>

<GlossaryCard 
  term="API Endpoint"
  category="Technical"
  definition={
    <>
      A specific URL where an API can access the resources it needs. Maxim AI provides various endpoints for different operations:
      <ul style={{ marginTop: '0.5rem', marginBottom: '0', paddingLeft: '1.5rem' }}>
        <li><code>/v1/chat/completions</code> - For chat-based interactions</li>
        <li><code>/v1/embeddings</code> - For generating embeddings</li>
        <li><code>/v1/models</code> - For listing available models</li>
      </ul>
    </>
  }
/>
