<GlossaryCard 
  term="Context Window"
  category="Technical"
  definition={
    <>
      The maximum number of tokens (words/characters) that can be processed in a single API request. Maxim AI supports context windows up to <strong>128K tokens</strong>. The context window includes both input and output tokens.
    </>
  }
/>

<GlossaryCard 
  term="Batch Request"
  category="Technical"
  definition={
    <>
      A method of sending multiple API requests together to improve efficiency and reduce latency. Batch requests allow you to:
      <ul style={{ marginTop: '0.5rem', marginBottom: '0', paddingLeft: '1.5rem' }}>
        <li>Process multiple operations in a single API call</li>
        <li>Reduce network overhead</li>
        <li>Improve overall performance</li>
      </ul>
    </>
  }
/>

<GlossaryCard 
  term="Backoff Strategy"
  category="Technical"
  definition={
    <>
      A retry mechanism that gradually increases the wait time between retry attempts when an API request fails. This helps prevent overwhelming the server and improves the chances of successful requests. Common strategies include:
      <ul style={{ marginTop: '0.5rem', marginBottom: '0', paddingLeft: '1.5rem' }}>
        <li><strong>Exponential backoff</strong>: Doubles wait time with each retry</li>
        <li><strong>Linear backoff</strong>: Increases wait time by a fixed amount</li>
      </ul>
    </>
  }
/>

<GlossaryCard 
  term="Rate Limit"
  category="Technical"
  definition={
    <>
      The maximum number of API requests allowed within a specific time period. Rate limits help:
      <ul style={{ marginTop: '0.5rem', marginBottom: '0', paddingLeft: '1.5rem' }}>
        <li>Prevent abuse and ensure fair usage</li>
        <li>Maintain system stability and performance</li>
        <li>Protect against DDoS attacks</li>
      </ul>
      <p style={{ marginTop: '0.75rem', marginBottom: '0' }}>
        Rate limits vary by plan. Check your plan details or contact support if you need higher limits.
      </p>
    </>
  }
/>

<GlossaryCard 
  term="Response Format"
  category="Technical"
  definition={
    <>
      The structure and format of data returned by API endpoints. Maxim AI typically returns JSON responses containing:
      <ul style={{ marginTop: '0.5rem', marginBottom: '0', paddingLeft: '1.5rem' }}>
        <li><strong>Content</strong>: The generated text or response</li>
        <li><strong>Metadata</strong>: Model information, tokens used, etc.</li>
        <li><strong>Usage statistics</strong>: Token counts and costs</li>
      </ul>
    </>
  }
/>

<GlossaryCard 
  term="Error Handling"
  category="Technical"
  definition={
    <>
      The process of anticipating, detecting, and responding to errors that occur during API requests. Effective error handling includes:
      <ul style={{ marginTop: '0.5rem', marginBottom: '0', paddingLeft: '1.5rem' }}>
        <li>Checking HTTP status codes</li>
        <li>Implementing retry logic with backoff</li>
        <li>Logging errors for debugging</li>
        <li>Providing user-friendly error messages</li>
      </ul>
    </>
  }
/>

<GlossaryCard 
  term="Deployment"
  category="Technical"
  definition={
    <>
      The process of making an AI model or application available for use in production environments. Maxim AI supports various deployment options:
      <ul style={{ marginTop: '0.5rem', marginBottom: '0', paddingLeft: '1.5rem' }}>
        <li><strong>API deployment</strong>: Access models via REST API</li>

        <li><strong>Custom endpoints</strong>: Deploy fine-tuned models as dedicated endpoints</li>
      </ul>
    </>
  }
/>

<GlossaryCard 
  term="Cost Optimization"
  category="Technical"
  definition={
    <>
      Strategies to reduce API usage costs while maintaining performance. Effective cost optimization includes:
      <ul style={{ marginTop: '0.5rem', marginBottom: '0', paddingLeft: '1.5rem' }}>
        <li>Using appropriate model sizes for tasks</li>

<GlossaryCard

  category="Technical"
